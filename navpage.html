Summary: Analysis of Airbnb Listings in San Francisco


Airbnb has been prey to controversies for multiple reasons in many different cities, especially in San Francisco. Over the time; city, its people, its government and Airbnb have learnt to live in harmony. This project was not about digging deep into contoversies. The main target of this project was to understand the spread of Airbnb listings in SF, study its pricing and create a simple predictive model to estimate the price of the listing if someone wanted to become an Airbnb host depending on few features of the property.

The intial idea was to scrape data from Airbnb website, but since Airbnb has been very infamous in past few years, I was lucky enough to find listings data from Tom Slee's website

    Here is the project summary:

  1. 1. Data Acquisition

    Data was collected from Tom Slee's website. Data was scraped once every month and I had some 20 csvs for 20 months. Data was not that accurate or perfect as it represented just one day of information for any given month.

  2. 2. Data Cleaning

    There were many extra spaces, special characters like '$', '{}' in the data and many null values for basic fields like number of bedrooms and bathrooms. Data was made model ready using Python libraries like Pandas and Numpy.

  3. 3. Feature and Model Selection

    There were number of features that were interesting to be used in a predictive model. Important question was to decide which predictive model to use. I started with Linear Regression and created few Univariate models. Those models were not at all robust and there was no correlation between price (target variable) and the univariate feature. I tried Multivariate Linear Regression then. Again the correlation was too low and model was not robust at all.

    Decided to move on from Linear Regression to Gradient Boosting as its more robust than Linear Regression. Started with 2 features (Number of bedrooms and bathrooms in potential listing) and noticed model score was better than Linear Regression and also the predictions made more sense. Included one more feature of neighborhood, which I thought was important and should have an impact on price of a airbnb listing.

    For the purpose of predictive modeling, data of only Oct 2017 was used in order to avoid the model to train on duplicate data of previous months.

  4. 4. Analysis and Dashboard creation

    I noticed many interesting things about Airbnb listings in SF. Some of them are displayed in Plots and Dashboard section. I used Tableau for visualization purpose.
